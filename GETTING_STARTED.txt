🚀 QUICK START (3 STEPS)

1️⃣  INSTALL DEPENDENCIES
   $ npm install

2️⃣  BUILD THE EXTENSION
   $ npm run build

3️⃣  LAUNCH IN VS CODE/CURSOR
   - Open this folder in VS Code/Cursor
   - Press F5
   - A new window opens with the extension activated

4️⃣  TEST IT WORKS
   - In the new window: Cmd+Shift+P (Mac) or Ctrl+Shift+P (Win/Linux)
   - Type: "Cursor Helper: Test Notification"
   - You should see a notification and hear a sound! 🎉

🎯 CURSOR INTEGRATION

Add a Cursor Rule to automatically trigger notifications:

📍 macOS/Linux Rule:
<!run:sh -lc "echo \"$(date) :: CURSOR_DONE\" > $HOME/.cursor-notify.flag">

📍 Windows Rule:
<!run:powershell -command "$d=Get-Date; Set-Content -Path $env:USERPROFILE\.cursor-notify.flag -Value $d;">

🎨 EXTENSIBILITY

The extension is designed for easy extension:

→ Add new watchers (IWatcher)
  - HTTP webhooks
  - Log file tailing
  - Process monitoring

→ Add new notifiers (INotifier)
  - Rich notifications with buttons
  - External webhooks
  - Email/Slack integration

→ Add new sound players (ISoundPlayer)
  - Custom audio formats
  - Text-to-speech

See CONTRIBUTING.md for implementation guides.

• View logs: View → Output → Select "Cursor Helper"
• Customize sound: Set "cursorHelper.customSoundPath"
• Adjust debounce: Set "cursorHelper.debounceMs"
